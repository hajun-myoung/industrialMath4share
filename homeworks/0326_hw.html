1번. CSS Selectors

CSS Selector는 기본적으로 CSS가 스타일링 할 요소를 선택할 수 있는 기술입니다

The CSS selectors module defines the patterns to select elements to which a set of CSS rules are then applied along with their specificity.

CSS selector 모듈은 스타일(CSS)을 씌우기 위한 요소(element)를 선택할 수 있는 패턴을 제공합니다.

-MDN Web Docs: CSS selectors

MDN 문서에서 설명해주는 것과 같이, CSS Selector의 특정한 "패턴"들을 사용하면, HTML 요소(Eleemnt)를 선택하는 것이 가능합니다. 이 때, 각각 다른 기호(또는 표현식)을 통해서 "어떤 요소를 특정"하는지가 달라집니다.

>, +, " "(스페이스 하나), ~

각각의 요소들은 MDN Web Docs 에서는, 다음과 같이 명명합니다

  1. > Child combinator
  2. " "(공백 한 칸) Descendant combinator
  3. + Next Sibling combinator
  4. ~ Subswquent sibling combniator

이 때, 각 요소는 다음과 같이 동작합니다:

Child combinator, ">"

자식 요소를 선택할 수 있게 합니다. 이 때, ">" 다음으로 지정된 요소(Element)는 반드시 "바로 오는 자식"이어야 합니다.

예를 들어,

            
<div class="box">
  <p class="child">
    자식1
    <p class="child">
      자식2
    </p>
  </p>
</div>

위와 같은 구조에서 .box > .child 를 사용하면 "자식2"가 쓰여있는 p 요소가 아닌, "자식1"이 쓰여있는 전체 p 요소를 특정합니다.

Descendant combinator, " "

공백 한 칸으로 특정되는 이 선택자는, > 과 그 기능이 "거의 동일"합니다.

다만, Child combinator와는 다르게 "바로 오는 자식"을 강제하지 않고, "부모-자식"관계이기만 하면 됩니다. 즉, .box .child라고 하면, box 클래스를 가진 요소 안 어딘가에 child 클래스를 가진 요소가 위치하면 되고, 따라서 Child combinator 에서 든 예시를 기준으로 "자식1"의 p요소와 "자식2"의 p요소가 "모두 각각 특정"됩니다

Next sibling combinator, +

이 선택자는 "부모-자식" 관계가 아닌, "형제" 관계를 특정합니다. 전통적인 트리 구조에서, 형제관계란 "같은 부모를 둔 관계"를 의미하므로, 마크업 언어인 html에서는 "같은 부모 요소" 안에 포함된 두 개 이상의 요소들을 "서로 형제 관계"라고 합니다. 

            
img + p {
  font-weight: bold;
}

이러한 선택자는, 위와 같이 사용할 때 가장 유용합니다. 해당 예시는 "모든 이미지 태그"의 "바로 다음에 오는 문단"을 굵게 처리합니다. 따라서, 이미지를 첨부하고 그에 대한 주석을 강조하기에 좋습니다

Subsequent-sibling combinator, ~

이 선택자는 Child combinator와 Descendant Combinator의 관계처럼, Next-sibling combinator와 달리 "이어서 오는 형제"이긴 하지만, "바로 다음에 올 것"을 요구하지는 않는 선택자입니다. 

          
img ~ p {
font-weight: bold;
}

즉, 위와 같이 구성하게 되면 img 태그 다음에 오는 "모든 문단"을 선택하여 특정합니다

2번. CSS의 세 가지 지정 방법

CSS에는 세 가지 지정 방법이 존재합니다

inline-CSS

첫 번째는 "태그 자체에 지정"하는 방법인 inline-css로,

      
<hr style="margin-top: 96px; color: lightgray" />

위와 같이, 태그 자체에 style 프로퍼티를 넣고, 이에 대한 값으로 CSS 문법으로 스타일을 지정합니다.

internal-CSS

두 번째는, "html 파일 내부에 CSS를 지정"하는 방법입니다. 이 경우,

      
<head>
  (...다른 헤드 태그들)
  <style>
    html {
      display: flex;
      justify-content: center;
    }
  </style>
</head>

위와 같이, head 태그의 자식으로 style 태그를 두고, CSS 문법을 사용해 스타일을 지정합니다. 이 때, 이 style 태그가 "전체 페이지의 CSS"를 특정하므로, CSS Selector의 사용이 필수적입니다.

이러한 방식을 사용하게 되는 경우 주의할 점이 하나 있는데, "다른 페이지"로 넘어가면 더 이상 적용되지 않는다는 점입니다. "다른 페이지"는 대개 "다른 html 파일"을 보여주는 행위이고, 따라서 특정 html 파일에 고정되어 있는 style 태그는, 다른 페이지에 영향을 주지 않습니다.

External-CSS

마지막으로, 가장 흔하게 사용되는 "외부 파일"을 이용한 방법입니다. 외부 파일이라고는 하지만, 대개 루트 경로보다 상위로는 접근할 수 없으며, 일반적으로 "html" 파일이 있는 경로, 또는 public 경로 정도에서 css 파일을 가져와 적용하게 됩니다. 

      
<head>
  (...다른 헤드 태그들)
  <link rel="stylesheet" href="index.css" />
</head>

위와 같이, link 태그를 이용해 파일을 연결하게 되며, 연결된 파일은 style 태그의 내부와 같이 CSS의 문법으로 작성됩니다. 이 때, href를 상대 경로로 동작합니다.

4번 display 속성의 종류와 기본 태그

display 속성에는 크게 세 가지의 종류가 있습니다. (이외에도 매우 다양한 속성이 존재하나, 가장 기본이 되는 속성이 세 가지입니다)

  1. 먼저, 한 줄을 통째로 차지하는 block 속성이 있으며, 이는 div 요소의 기본값입니다
  2. 다음으로, 다른 요소들과 줄을 같이 사용하는, "한 줄에 들어가는" inline 속성이 있으며, 이는 span 요소의 기본값입니다
  3. 마지막으로, 보이지 않게 되는 none 속성이 있으며, 이를 기본으로 하는 태그는 존재하지 않습니다.

padding과 margin

이 두 속성값은 모두 "여백"을 설정합니다. 그런데, padding "요소 내부에 채워지는 방식"으로 동작하고, margin은 "요소 외부에 여백을 두는 방식"으로 동작합니다

margin
padding

즉, 위와 같이 주황색 div를 기준으로 외부영역은 주황색 div의 margin, 내부 영역은 주황색 div의 padding에 해당합니다

margin collapsing

이 때, margin에 대해서는 html에 특수한 성질이 적용되어, 주의가 필요합니다

세로로 배치된 요소들이 둘 다 margin을 가지고 있을 때, "비교적 위에 있는 요소"의 margin-bottom은, "비교적 아래 있는 요소"의 margin-top과 병합되어 처리합니다

즉, 위 요소가 margin: 50px;를 가지고 있고, 아래 요소가 margin: 50px;를 가지고 있을 때, 이 두 요소는 100px가 아닌 "병합된 50px" 만큼의 여백만을 가지고 있습니다. 이와 같은 원리로 인하여, 위 padding-margin을 설명하는 그림을 그릴 때, 부모 div에 padding을 주어, "내부 요소의 margin"이 있는 것 처럼 연출한 것을 볼 수 있습니다.
